Design Folder Organization

The goal is simple: keep capture effortless, keep current work visible, and stop forcing every document into a type-based taxonomy.

Why the Old Structure Creates Friction

The document says the problem is not a lack of folders. It is too much precision too early.

Mixed dimensions

Folders combine content type, life area, and intent, so every filing decision becomes interpretive work.

Micro-decisions everywhere

Notes versus snippets versus reference sounds tidy, but it creates hesitation at the moment of capture.

Knowledge turns into a junk drawer

When reusable material has one vague home, retrieval gets worse instead of better.

Inbox already proves the point

The current inbox works because it is low-friction, fast, and decision-light.

Organize by Function, Not Type

Each top-level folder should answer one question only.

Inbox Where does this go right now?
Active What am I currently thinking about?
Projects What has an outcome?
Areas What responsibilities never really end?
Library or Reference What do I mainly want to retrieve later?
Archive What is done or irrelevant now?

Recommended Work Structure

For the work reorg, the document recommends this shape.

design/
        ├── inbox/
        ├── active/
        ├── projects/
        ├── areas/
        ├── reference/
        └── archive/

The big change is active/

Current initiatives, messy planning, perf investigations, and half-finished thinking get a first-class home.

What Goes Where in Work

The key move is to sort by purpose rather than file type.

inbox/ Fast capture with zero thinking.
active/ Current initiatives, on-call issues, and tactical in-progress work.
projects/ Deliverable-driven efforts such as launches, migrations, decks, and named initiatives.
areas/ Career, engineering, operations, and strategy live here because they do not end.
reference/ Static or reusable docs like guides, architecture notes, and AI material.
archive/ Finished projects and superseded context that should leave the working set.

Rule Changes That Make It Work

The folder names matter, but the operating rules matter more.

Default destination is active/

When you are unsure, put it with current thinking instead of chasing a perfect category.

Stop distinguishing notes, ideas, and snippets

They are all documents. Let naming and search carry the meaning.

Flatten the tree

The guidance is to reduce depth aggressively and keep the structure close to two levels when possible.

Projects contain everything

No notes folder, ideas folder, or drafts folder inside the project. If it belongs, it stays there.

Personal vs Work Naming

The role is similar in both systems, but the document intentionally uses different names.

Personal uses library/

Broader, more idea-heavy, and better suited to a creative personal system.

The personal tree emphasizes creativity, expression, and low-friction exploration.

Work uses reference/

Practical, reusable, and easier to scan in a work context.

The document explicitly says this rename matches how work brains think.

Weekly Inbox Reset

The maintenance ritual is intentionally tiny so the system can survive real life.

Open inbox Start from the capture pile.
Still active? Move it to active/.
Has an outcome? Move it to projects/.
Reference or delete If it is reusable, file it. Otherwise, remove it.

End State

This reorg keeps what already works and simplifies everything after capture.

Fast capture inbox/ stays easy and trusted.
Low-pressure thinking active/ gives unfinished work a real home.
Cleaner retrieval projects/, areas/, and reference/ each have one job.
