Two-Tier Bookmark System

A guide to a two-tier bookmark system: the Bookmark Bar for active links (RAM) and the archive for searchable storage (Hard Drive).

The Two-Tier System

Treat your browser bookmarks like short-term memory and your bookmark manager like long-term storage.

Tier 1: Bookmark bar

For active, high-frequency links. This is your clickable workspace.

Tier 2: Archive

For searchable long-term storage. Tag every entry with context so search stays scoped.

Personal Stack: Bookmark Bar

This is your clickable workspace. Organize icons by muscle memory and folders by workflow.

🧭 Icon Row Personal email, Hacker News, YouTube, Pandora, the archive, and communities stay as icon-only pins so the bar stays wide open.
📥 Inbox Drop anything you truly plan to revisit in the next few days, then clear it every Sunday morning over coffee.
🌅 Routine One click opens weather, news, finance, or journaling tabs so mornings start in the same groove.
✈️ InFlight A rotating folder for whatever deserves a sprint right now—trip planning, a big purchase, or a personal code project.

Personal Stack: Archive

Anything that is not actionable this week belongs down here.

🏗 Projects Archive Personal projects such as renovations, health plans, or coursework live here once the sprint is over.
📚 Reference Long-form learning resources and deep-dive tutorials stay searchable without crowding the bar.
🧰 Tool Library Experiments, progressive web apps, and online utilities you want to remember later.
🛒 Home Shopping lists, recipes, household management, finances, and other life-admin docs.
🗄 Archive Older seasons of life and bulk imports; keep nesting shallow so search keeps working.

Work Stack: Bookmark Bar

Muscle-memory links stay up top so you can reset your workspace in seconds and see only the sprint in front of you.

🧭 Icon Row Email, Calendar, Slack, Jira, GitHub, and similar systems are pinned as icons for fast muscle-memory clicks.
🌅 Startup Middle-click one folder each morning to open Jira, inbox, calendar, and monitoring tabs in sequence.
📥 Inbox Temporary tickets, PRs, or research notes live here; clear it every Friday at 4:30 PM so the bar never becomes storage.
🗂 Planning Links to plans, spreadsheets, decks, or Miro boards that actively steer the work.
🛠 Tools CI/CD consoles, cloud dashboards, log viewers, and Linear/Jira search views you hop into constantly.
✈️ InFlight One folder per engagement. Move it to the archive as soon as the work is signed off.
📚 Docs Framework docs, internal wikis, or API specs that you reference weekly.
📊 Dashboards Grafana, Looker, or BI boards that give an instant health check.

Work Stack: Archive

Everything in the archive is searchable without touching the bar.

🏗 Projects Archive Project research and engagement-specific resources once the daily grind is done.
📚 Reference Playbooks, canonical docs, and API references too valuable to lose.
🧰 Tool Library Infra consoles, experiments, or scripts that you do not need on the bar but want one search away.
🏢 Company HR, benefits, org charts, policy docs, and anything that keeps company ops separate from engineering flow.
🗄 Archive Past projects and deprecated docs filed by year or project so the working set stays tiny.
